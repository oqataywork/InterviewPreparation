Интерфейс - позволяет определить некоторый функционал(методы и свойства, члены должны быть публичными), не имеющий конкретной реализации. (Контракт)

Абстрактный класс - базовый класс, который не предполагает создания экземпляров и может содержать в себе нереализованные члены.
Ключоевое слово abstract

Использование абстрактных классов:
-общий функционал для родственных объектов
-необходимость в базовой функциональности для дочерних объектов

Использование интерфейсов:
- Необходимо определить функционал, который будет необходимо реализовать для группы разрозненных объектов
- Когда есть необходимость определить небольшой тип
